# Specs Index

This folder contains lightweight product/reference specs for the Databricks Associate Study App.

Use these docs when you need intent and scope context before making changes.

## Files

- [`application-goal.md`](./application-goal.md): Core mission, target user, and what success looks like for this app.
- [`feature-scope.md`](./feature-scope.md): Current feature boundaries, non-goals, and implementation constraints for v1.

## How agents should use this folder

1. Start with `application-goal.md` to understand the "why" behind the product.
2. Review `feature-scope.md` to confirm whether a requested change aligns with current scope.
3. If there is a conflict between a request and these docs, call it out explicitly in your summary/PR notes.
